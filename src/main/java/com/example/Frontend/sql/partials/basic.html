Basic SQL Queries

Structured Query Language (SQL) provides a set of commands to interact with the data stored in relational databases. These commands allow you to retrieve, insert, update, and delete records. They form the foundation of working with databases and are essential for developers, analysts, and database administrators alike.

1. Retrieving Data — SELECT

The SELECT statement is used to fetch data from one or more tables. You can retrieve all columns or only specific ones, and filter results based on conditions. 

-- Retrieve all columns from the "users" table
SELECT * FROM users;

-- Retrieve specific columns (name and email)
SELECT name, email FROM users;

-- Retrieve only active users
SELECT * FROM users
WHERE status = 'active';

-- Retrieve users sorted by registration date (newest first)
SELECT * FROM users
ORDER BY registration_date DESC;

Tip: Use LIMIT to restrict the number of results returned, especially with large datasets.

2. Adding Data — INSERT

The INSERT statement is used to add new rows into a table. You can insert a single row or multiple rows in one statement. 

-- Insert one row into the "users" table
INSERT INTO users (name, age, email)
VALUES ('John Doe', 30, 'john@example.com');

-- Insert multiple rows at once
INSERT INTO users (name, age, email)
VALUES 
('Jane Smith', 25, 'jane@example.com'),
('Alex Brown', 28, 'alex@example.com');

Note: Always specify the column names to avoid errors if the table structure changes in the future.

3. Updating Data — UPDATE

The UPDATE statement modifies existing rows in a table. It’s important to include a WHERE clause to avoid updating every row unintentionally. 

-- Update a single user's age
UPDATE users
SET age = 31
WHERE name = 'John Doe';

-- Give a 10% salary increase to all employees in the "Engineering" department
UPDATE employees
SET salary = salary * 1.10
WHERE department = 'Engineering';

Warning: Without a WHERE clause, the update will apply to every row in the table.

4. Deleting Data — DELETE

The DELETE statement removes one or more rows from a table. As with UPDATE, always use a WHERE clause unless you intend to remove all data. 

-- Delete users younger than 18
DELETE FROM users
WHERE age < 18;

-- Delete all orders that have been cancelled
DELETE FROM orders
WHERE status = 'cancelled';

Tip: Use TRUNCATE TABLE to quickly remove all data from a table — it’s faster but cannot be rolled back in some databases.

5. Filtering Results — WHERE

The WHERE clause allows you to filter data based on conditions. It supports operators such as =, !=, <, >, LIKE, and IN. 

-- Find users with age over 25
SELECT * FROM users
WHERE age > 25;

-- Find users whose name starts with "A"
SELECT * FROM users
WHERE name LIKE 'A%';

-- Find orders placed by specific customers
SELECT * FROM orders
WHERE customer_id IN (101, 102, 103);

6. Sorting Results — ORDER BY

-- Sort users by name in ascending order
SELECT * FROM users
ORDER BY name ASC;

-- Sort users by age in descending order
SELECT * FROM users
ORDER BY age DESC;

7. Limiting Results — LIMIT / OFFSET

-- Retrieve only the first 10 rows
SELECT * FROM products
LIMIT 10;

-- Retrieve rows 11 to 20
SELECT * FROM products
LIMIT 10 OFFSET 10;

8. Combining Filters

-- Find all active users above age 25
SELECT * FROM users
WHERE status = 'active' AND age > 25;

-- Find all products in category 5 OR priced under $50
SELECT * FROM products
WHERE category_id = 5 OR price < 50;

Conclusion

These basic SQL queries form the foundation of database interaction. By mastering them, you’ll be able to perform most day-to-day database operations. As you progress, you’ll learn more advanced topics like JOINs, GROUP BY, subqueries, aggregate functions, and transactions that allow you to handle more complex scenarios.
